Controllers

Gestion des contrôleurs

Création de contrôleur

Pour créer un contrôleur, vous devez vous rendre dans le dossier src puis controller de votre projet. C'est à ce niveau que vous créerez vos fichiers PHP en respectant la convention de nommage des contrôleurs c’est à dire en ajoutant .class avant .php.

src 
    > controller 
        > votre-fichier.class.php

Définition du code des contrôleurs

Héritage

Chaque contrôleur créé doit hériter de la super class controller du framework.

N'oubliez pas d'effectuer l'héritage entre constructeur.

<?php
use libs\system\Controller;
class Test extends Controller{
    public function __construct(){
        parent::__construct();
        //Appel du model grâce au systeme d'autoloading
    }
}
?>

Chargement des vues

Le chargement des vues à partir du contrôleur est simple. On utilise l'instruction suivante :

<?php
    ...
    return $this->view->load("test/index");
    ...
?>

Vous devez à présent créer le fichier index.html, pour le chargement de la vue, dans le dossier testqui sera lui aussi créer dans src > view.

<?php
    public function index(){
        return $this->view->load("test/index");
    }
?>

Chargement des modèles

La couche modèle permet de récupérer ou de mettre à jour des données de la base de données. Les modèles sont des classes qui héritent de la classe modèle du framework (voir la partie modèle).

Pour les chargements vous pouvez utiliser le mot clé USE si un namespace a été précisé au niveau du modèle. Puis vous n’avez qu’à instancier et commencer à utiliser votre modèle.
